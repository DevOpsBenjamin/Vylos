Release Notes — @vylos/core + @vylos/cli

========================================
v1.2.1
========================================

BREAKING CHANGES:
- Internal UI types (DialogueState.text, ChoiceOption.text, ActionEntry.label,
  LocationEntry.name, DrawableEventEntry.label) changed from `string` to `TextEntry`.
  Components that override DialogueBox, ChoicePanel, ActionOverlay, LocationOverlay,
  or DrawableOverlay must now call `resolveText()` from `useLanguage()` on these fields.
- EventRunnerCallbacks: `resolveText` replaced by `normalizeText` (string|TextEntry → TextEntry).
  `onSay` signature: (text: TextEntry, speaker, variables?) instead of (text: string, speaker).
  `onChoice` signature: (options: ChoiceOption[]) — ChoiceOption.text is now TextEntry.
- EventManager.getDrawableEvents() now requires a `normalizeText` parameter.

NEW:
- Full reactive i18n: language changes in settings instantly update ALL displayed text
  (dialogue, choices, actions, locations, drawable events, history) without navigation.
- DialogueState.variables: interpolation variables stored separately, resolved at render time.
- setGlobalLanguage() exported from @vylos/core for override authors.

FIXES:
- Language change in settings had no effect (SettingsManager → LanguageManager not wired)
- Language change required navigation to update overlays (pre-resolved strings in store)
- History browsing showed text in the language it was originally displayed in, not current language

========================================
v1.2.0
========================================

BREAKING CHANGES:
- VylosConfig.languages changed from `string[]` to `LanguageEntry[]`
  Before: languages: ['en', 'fr']
  After:  languages: [{ code: 'en', label: 'English' }, { code: 'fr', label: 'Français' }]
  Migration: Update vylos.config.ts in all projects.

NEW:
- LanguageEntry type exported from @vylos/core ({ code: string, label: string })
- SettingsMenu language selector now reads labels from config (no more hardcoded English-only)

FIXES:
- SettingsMenu: language dropdown was hardcoded to English only, now dynamically populated from config.languages

INTERNAL:
- Template vylos.config.ts updated to use LanguageEntry format
- All sample projects (1-basic, 2-advanced, 3-phone) updated

NOTE: External consumer projects (e.g. Mina Alchemist) must update their vylos.config.ts
